Assignment Plan

1. Choose a Software Application (1 day)

  • Time: 1 day
  • Activities:
    • Choose an e-commerce website as the software application.
    • Outline main features and functionalities.

2. Develop the E-Commerce Website (5 days)

  • Time: 5 days
  • Activities:
    • Day 1: Set up the development environment and project structure.
    • Day 2: Implement user registration and login functionalities.
    • Day 3: Implement product search, filtering, and product listing.
    • Day 4: Implement add to cart and checkout process.
    • Day 5: Implement payment processing and order history.

3. Write Detailed Test Cases (3 days)

  • Time: 3 days
  • Activities:
    • Day 1: Identify at least 10 key functionalities.
    • Day 2: Write detailed test cases for the first 5 functionalities.
    • Day 3: Write detailed test cases for the remaining functionalities and review.

4. Raise Bugs and Create Task Tickets in JIRA (3 days)

  • Time: 3 days
  • Activities:
    • Day 1: Execute test cases for the first 5 functionalities and document any bugs.
    • Day 2: Execute test cases for the remaining functionalities and document any bugs.
    • Day 3: Create JIRA tickets for each bug and additional tasks.

5. Automation of Specific Flows (4 days)

  • Time: 4 days
  • Activities:
    • Day 1: Choose at least two workflows to automate (e.g., Login Flow, Checkout Flow).
    • Day 2: Write automated test scripts for the Login Flow.
    • Day 3: Write automated test scripts for the Checkout Flow.
    • Day 4: Review and test the automated scripts, ensuring they run successfully.

6. Performance Testing (2 days)

  • Time: 2 days
  • Activities:
    • Day 1: Set up the performance testing environment and create a test plan.
    • Day 2: Execute the performance tests and document the results.

7. Submission (2 days)

  • Time: 2 days
  • Activities:
    • Day 1: Compile all test cases, bug reports, performance test results, JIRA screenshots, and automated test scripts into a single document.
    • Day 2: Prepare the source code in a GIT repository and review the final document for submission.

Daily Breakdown and Prioritization

Week 1:

  • Day 1: Choose e-commerce website as the software application.
  • Day 2-6: Develop the e-commerce website.
  • Day 7: Write detailed test cases (Day 1).

Week 2:

  • Day 8-10: Write detailed test cases (Days 2-3).
  • Day 11-13: Execute test cases and raise bugs in JIRA.
  • Day 14-17: Automate specific flows (Login and Checkout).
  • Day 18-19: Perform performance testing.
  • Day 20: Compile and submit the final document.

Tips for Efficient Completion

  • Time Management: Allocate specific hours each day to focus solely on the assignment.
  • Collaboration: If possible, work with a peer to review test cases and scripts for accuracy.
  • Automation Tools: Familiarize yourself with automation and performance testing tools in advance to save time during implementation.
  • Documentation: Keep detailed notes and document each step as you go to make the final compilation easier.
